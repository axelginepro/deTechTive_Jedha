1. Contexte & Objectifs

Scénario : Pilotage d'un projet d'infrastructure global : création de l'environnement réseau et intégration complète de la stack technique (Web, Data, Fichiers).

  • Confidentialité : Chiffrement intégral des flux (HTTPS/SSL) et sécurisation du stockage par cloisonnement (ACL).
  • Disponibilité : Une architecture robuste : grâce au cloisonnement (VLAN), la panne d'un élément ne coupe pas tout le système.
  • Traçabilité : Centralisation des logs (Wazuh) et authentification unique (AD).

2. Architecture Infrastructure

Topologie GNS3

  • VLAN 10 : Zone de Management (Wazuh, Admin).
  • VLAN 20 : Zone Serveurs (AD, Web, Files, BDD).
  • VLAN 30 : Zone Utilisateurs (Workstations).

Plan d'Adressage

Cliquez pour voir les plages IP et flux
  • Segmentation : Filtrage strict par pare-feu pfSense.
  • Isolation : La BDD est confinée au VLAN 20. Tout accès direct depuis les VLANs 10 et 30 est bloqué.

3. Gestion des Identités (Active Directory)

Moindre Privilège (Least Privilege)

Le principe fondamental de sécurité appliqué au projet.

  • Structure AD : Organisation par Unités d'Organisation (OU) pour séparer les Agents des Administrateurs.
  • Droits Restreints : Les utilisateurs standard n'ont aucun droit d'administration sur les serveurs ou les postes de travail.
  • Comptes de Service : L'application Web utilise un compte AD dédié avec des droits minimaux, uniquement capable d'écrire dans les dossiers de dépôt.

Serveur de Fichiers Sécurisé

Isolation stricte des données sensibles.

  • Dossiers Dédiés : Chaque agent (ou équipe) dispose d'un dossier partagé personnel sur le File Server.
  • Permissions NTFS : Configuration des ACLs (Access Control Lists) pour que l'Agent A ne puisse techniquement ni lire ni voir le dossier de l'Agent B.
  • Mappage Dynamique : Au login, seul le lecteur réseau correspondant à l'utilisateur connecté est monté.

4. Développement & Backend (Détail)

Apache PHP 8.x MariaDB Windows Server (SMB)

Serveur Web (Front)

Le point d'entrée unique.

  • Rôle Pivot : Il orchestre les demandes entre l'utilisateur, la BDD et le File Server.
  • Sessions : Gestion stricte via session_start(). Redirection si non authentifié.
  • Sécurité : Validation des entrées (Input Sanitization) et whitelist des fichiers uploadés.

Base de Données

Le coffre-fort des métadonnées.

  • Connexion SSL : Le client PHP utilise un certificat CA pour valider l'identité du serveur MariaDB.
  • Isolation : Hébergée sur une VM dédiée (192.168.10.18), inaccessible du public.
  • Failover : Code PHP avec try/catch pour basculer en mode dégradé si le SSL échoue.

File Server (SMB)

Stockage des preuves.

  • Protocole : Partage Windows (SMB) monté dynamiquement.
  • Bridge PHP : L'application utilise la commande net use pour monter le lecteur réseau à la demande.
  • Permissions : Utilisation d'un compte de service AD spécifique (Least Privilege) pour l'écriture.

5. Sécurité Applicative (DevSecOps)

Protection du Code

  • Anti-SQLi : Usage exclusif de PDO et des requêtes préparées.
  • Anti-RCE : Renommage forcé des fichiers uploadés (preg_replace) pour éviter l'exécution de scripts.
  • Feedback : Indicateur visuel "Cadenas Vert/Rouge" informant l'agent de l'état du chiffrement.

Sécurité Infrastructure

  • Pare-feu : pfSense filtre tout trafic non légitime entre les VLANs.
  • SIEM : Wazuh remonte les alertes de sécurité en temps réel.
  • Hardening : Désactivation des services inutiles et principe du moindre privilège.
